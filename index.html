Hieroglyphic encoding (RES) in web pages using JavaScript

This package offers an elegant framework for rendering hieroglyphic in web pages, using JavaScript. Other technologies used are HTML5 Canvas and TrueType. No server-side computation is involved.

One can use RES or RESlite. Rendering of RESlite is fast, but requires compilation from RES, for instance using PhilologEg. RES is easier to use and more powerful, but can be slow to render for web pages that include large amounts of hieroglyphic text.

This package includes:

  • test suite for RESlite
  • test suite for RES
  • skeleton of web page with RESlite only
  • skeleton of web page with RES/RESlite

Download and instructions for use

  1. Download: resjs.tar (approx. 950KB).
  2. Unpack it.
  3. To use RESlite only, one can create new web pages on the basis of skeleton_lite.html. One needs to include with the web pages all the files ending on .ttf and .css, as well as res_aux.js, res_lite.js, res_web.js.
  4. To use any RES/RESlite encodings, one can create new web pages on the basis of skeleton_res.html. One needs to include with the web pages all the files ending on .ttf, .js, and .css.

Frequently asked questions

For which browsers is this intended?

In principle this should run on all modern browsers. So far it has been tested on Chromium 34, Firefox 28, Rekonq 2.4.2.

Why don't you use SVG?

We tried it, using both Raphael and Snap. In both cases the path intersection operation that we need for the implementation of e.g. insert is far too slow to be useable in practice. A rendered HTML5 Canvas is not scalable, but the rendering process is parameterizable by a font size, so we don't see this as a problem for most web applications.

Who can use this software?

Anyone can use, modify and redistribute the JavaScript any way they like. Acknowledgements where appropriate will of course be appreciated. For the font, see the pages of NewGardiner.
